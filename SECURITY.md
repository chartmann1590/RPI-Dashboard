# Security Policy

If you discover a security vulnerability, please follow these steps:

Do not open a public issue

Open a private security advisory or contact the maintainers directly

Provide details about the vulnerability

Allow time for the issue to be addressed before public disclosure

List which versions are currently supported with security updates

Security Best Practices

Based on the project, any security considerations users should be aware of (like protecting API keys, environment variables, etc.)

Disclosure Policy

How security issues will be handled and disclosed.
